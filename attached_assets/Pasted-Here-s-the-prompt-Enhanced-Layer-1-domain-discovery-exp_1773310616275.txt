Layer 1 — Enhanced domain inference with content validation
For each lead, generate candidate domains using these slug patterns in order. Try all patterns before giving up, but stop as soon as one passes content validation:
Slug generation (apply to the cleaned company name — strip Ltd, Limited, LLP, PLC, & Co, And Co, Group, UK, (UK), punctuation):
	1.	Full name hyphenated: the-smith-consulting-group.co.uk / .com / .uk
	2.	Full name no separator: thesmithconsultinggroup.co.uk / .com
	3.	Significant words only (drop words under 3 characters — of, at, in, the, and): smith-consulting-group.co.uk / .com
	4.	Significant words no separator: smithconsultinggroup.co.uk / .com
	5.	First two significant words only: smithconsulting.co.uk / .com
	6.	Initials of all significant words: scg.co.uk / .com — only attempt if initials string is 3+ characters to avoid false positives
	7.	First significant word only: smith.co.uk / .com — only attempt if that word is 6+ characters
For each candidate, first send a HEAD request (3s timeout). If it resolves (200, 301, 302), proceed to content validation before accepting.
Content validation (mandatory for every Layer 1 hit)
Fetch the homepage (GET, 8s timeout, standard headers). Then check all of the following:
	1.	Name match — does the page title, H1, or visible body text contain at least one significant word from the company name? If none appear anywhere on the page, reject.
	2.	Parked domain check — does the page contain any of: "domain for sale", "buy this domain", "parked by", "coming soon", "under construction", "godaddy", "sedoparking", "hugedomains"? If yes, reject.
	3.	Location plausibility — scan for GU postcode pattern (GU\d) or any of: Surrey, Godalming, Guildford, Farnham, Woking, Haslemere. If found, mark geo_signal = "confirmed". If not found, mark geo_signal = "none" — do not reject on this basis alone, just flag it.
	4.	Minimum content threshold — if the page body text is under 100 words, flag as thin_content = true but do not reject automatically.
Confidence scoring — assign each validated hit a domain_confidence value:



|Condition                                                   |Score   |
|------------------------------------------------------------|--------|
|Name match + location confirmed + full slug pattern (1 or 2)|`high`  |
|Name match + location confirmed + shortened slug (3–5)      |`high`  |
|Name match + no location signal + full slug                 |`medium`|
|Name match + no location signal + shortened slug            |`medium`|
|Name match + thin content                                   |`low`   |
|Initials or first-word-only slug + name match               |`low`   |

Only accept high and medium confidence hits automatically. For low confidence hits: store the domain in a separate field domain_candidate but leave website blank and append low_confidence_domain to enrichment_source — do not use these for email generation.
